def parse_timings(raw: str) -> dict[str, float]:
    out: dict[str, float] = {}
    for key, pats in _PATTERNS.items():
        for pat in pats:
            m = re.search(pat, raw)
            if m:
                out[key] = float(m.group(1))
                break
    if "prompt_ms" in out and "prompt_tokens" in out and out["prompt_ms"] > 0:
        out["prefill_tps"] = out["prompt_tokens"] / (out["prompt_ms"] / 1000)
    if "eval_ms" in out and "eval_tokens" in out and out["eval_ms"] > 0:
        out["decode_tps"] = out["eval_tokens"] / (out["eval_ms"] / 1000)
    if "load_ms" in out and "prompt_ms" in out:
        out["ttft_ms"] = out["load_ms"] + out["prompt_ms"]
    return out

ABSTRACT:
We present RotorQuant, a reimagining of TurboQuant (ICLR 2026) that replaces the d×d random orthogonal rotation matrix with Clifford rotors R = exp(B/2) in the geometric algebra Cl(3,0). Instead of a matrix multiply requiring d² multiply-adds, RotorQuant performs the rotor sandwich product RxR̃ using only ~100 multiply-adds per vector, exploiting rotor sparsity (4 of 8 multivector components are zero). Fused GPU kernels (CUDA for NVIDIA, Metal for Apple Silicon) implementing the full pipeline (embed, rotor sandwich, Lloyd-Max quantize, inverse sandwich, extract) achieve 10-19x speedup on NVIDIA and 9-31x speedup on Apple Silicon over TurboQuant's BLAS matmul, while using 44x fewer parameters (372 vs 16,399 for d=128). Validated on real KV cache data from Qwen2.5-3B-Instruct, RotorQuant matches TurboQuant's attention fidelity (cosine similarity 0.990 vs 0.991) and achieves higher top-1/top-5 retrieval accuracy at 4K context. Code and benchmarks: https://github.com/scrya-com/rotorquant

TORQ es la técnica "v-torque" (Vector TORQ) del superpool.
Añade una capa de cuantización de activaciones MXFP4 que:
- Se fusiona con los pesos de la capa lineal adyacente (overhead ~0 en inferencia)
- Solo necesita 128 muestras de calibración
- Es training-free (PTQ)
- Ideal para el perfil **max-power** cuando se busca máxima calidad

def save_run(self, run: Run) -> int:
        cur = self._conn.execute(
            "INSERT INTO runs (cell_key,n_reps,raw_ref,error,created_at) VALUES (?,?,?,?,?)",
            (run.cell_key, run.n_reps, run.raw_ref, run.error,
             time.strftime("%Y-%m-%dT%H:%M:%S")))
        run_id = cur.lastrowid
        self._conn.executemany(
            "INSERT INTO measurements (run_id,metric,value,unit,run_index) VALUES (?,?,?,?,?)",
            [(run_id, m.metric, m.value, m.unit, m.run_index) for m in run.measurements])
        self._conn.commit()
        return int(run_id)
